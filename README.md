# Покерный таймер — Клубный турнир

Таймер блайндов для домашнего/клубного покера: структура уровней, ребаи, аддоны,
перерывы, случайная жеребьёвка мест и дилера, расчёт призовых и «кто кому платит»,
ростер клуба, архив турниров и таблица сезона. Ар-деко оформление (шрифт Cinzel),
две темы, работает офлайн (PWA), с опциональной синхронизацией через Supabase.

Приложение статическое — **без сборки**. Можно открыть `app/index.html` напрямую или
задеплоить папку `app/` на любой статический хостинг (например, Vercel).

## Структура

```
app/            — само приложение (открывается в браузере, работает офлайн)
  index.html
  app.js style.css
  fonts/        — самохостящийся шрифт Cinzel (офлайн)
  vendor/       — офлайн-клиент Supabase
  config.js     — сюда впишите ключи Supabase, чтобы включить синхронизацию
  sw.js manifest.json icon.svg  — PWA (установка на домашний экран)
supabase/schema.sql  — схема БД + доступы (выполнить один раз в Supabase)
vercel.json          — конфиг деплоя (папка приложения — app/)
SUPABASE_SETUP.md    — пошаговая настройка синхронизации
```

## Запуск локально

Просто откройте `app/index.html` в браузере. Для входа по ссылке (синхронизация)
нужен локальный сервер на фикс-порту — см. SUPABASE_SETUP.md.

## Деплой на Vercel

1. Запушьте этот репозиторий на GitHub.
2. Vercel → Add New → Project → импортируйте репозиторий → Deploy.
   (Vercel подхватит `vercel.json`; каталог приложения — `app/`.)
3. Для синхронизации: заполните `app/config.js` (URL + anon-ключ Supabase) и настройте
   проект по `SUPABASE_SETUP.md`. В Supabase → Authentication → URL Configuration
   пропишите адрес Vercel в **Site URL** и **Redirect URLs**.

Синхронизация необязательна — без неё приложение полностью работает локально.
