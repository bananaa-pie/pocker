# Облачная синхронизация — настройка (Supabase + Vercel)

Приложение работает **полностью локально** и без облака. Эта инструкция включает
синхронизацию ростера и архива турниров между устройствами. Всё бесплатно.

Порядок: сначала Supabase (база + вход по email), потом впишем ключи в `config.js`,
потом (по желанию) деплой на Vercel, чтобы открывать приложение по ссылке.

---

## 1. Создать проект Supabase (~3 мин)

1. Зайдите на **https://supabase.com** → Sign in (можно через GitHub).
2. **New project**. Название любое, задайте пароль базы (запишите), регион — ближе к вам.
3. Подождите ~1–2 минуты, пока проект поднимется.

## 2. Создать таблицы (~1 мин)

1. Слева **SQL Editor** → **New query**.
2. Откройте файл `supabase/schema.sql` из этого проекта, скопируйте **всё** содержимое в редактор.
3. Нажмите **Run**. Должно выполниться без ошибок (создаст таблицы `clubs`,
   `club_members`, `tournaments`, политики доступа и функции входа).

## 3. Включить вход по email-коду (~1 мин)

1. Слева **Authentication → Sign In / Providers → Email**.
2. Убедитесь, что **Email** включён.
3. Рекомендуется **выключить** «Confirm email» / включить OTP-код, чтобы вход был по
   6-значному коду из письма (а не по ссылке). В разделе Email включите **Email OTP**.
   (Если оставить только magic-link — вход тоже сработает, но код удобнее в PWA.)

> Бесплатный лимит писем Supabase небольшой, но для домашней игры его хватает с запасом.

## 4. Взять ключи и вписать в `config.js` (~1 мин)

1. Слева **Project Settings → API**.
2. Скопируйте:
   - **Project URL** → это `url`
   - **anon public** ключ (раздел Project API keys) → это `anonKey`
3. Откройте `app/config.js` и впишите их:

   ```js
   window.PK_SUPABASE = {
     url: 'https://ВАШ-ПРОЕКТ.supabase.co',
     anonKey: 'eyJhbGciOi....(длинная строка)',
   };
   ```

> Эти значения **не секретные** — `anon`-ключ предназначен для браузера, а доступ к
> данным ограничивают RLS-политики из `schema.sql` (видно только свой клуб).
> **Не** используйте здесь `service_role` ключ.

Готово. Откройте `app/index.html` → экран **♣ Клуб** → впишите email → введите код из
письма. После входа ростер и архив уедут в облако и подтянутся на любом устройстве,
где вы войдёте тем же email.

---

## 5. (По желанию) Деплой на Vercel — открывать по ссылке

1. Залейте этот репозиторий на GitHub.
2. На **https://vercel.com** → **Add New → Project** → импортируйте репозиторий.
3. Vercel подхватит `vercel.json` (папка приложения — `app`). Нажмите **Deploy**.
4. Получите ссылку вида `https://ваш-проект.vercel.app` — открывается на телефоне,
   ставится на домашний экран (PWA), работает офлайн.

> `config.js` уже содержит ваши (несекретные) ключи и задеплоится вместе с фронтом —
> отдельные переменные окружения не нужны.

### Про redirect (если оставили magic-link вместо OTP)
Если вход по ссылке: в Supabase **Authentication → URL Configuration** добавьте адрес
Vercel в **Site URL** и **Redirect URLs**. При входе по коду (OTP) это не требуется.

---

## Как это работает (коротко)

- **Local-first.** Основное хранилище — браузер (localStorage). Таймер и весь турнир
  работают без сети. Облако — слой поверх: вошли → данные слились (union) и
  выгрузились; нет сети → просто работаете локально, синк произойдёт позже.
- **Один аккаунт = один клуб.** Входите тем же email на разных устройствах — те же
  данные. **Код клуба** (виден на экране Клуб) нужен, только если доступ к вашему клубу
  открывает **другой человек**: он вводит код в поле «Подключить клуб».
- **Модель данных.** `clubs.roster` — ростер (JSON); `tournaments` — по строке на
  турнир (id совпадает с локальным, поэтому дубликатов при слиянии не будет). Таблица
  сезона считается на клиенте из архива.
- **Безопасность.** RLS-политики пускают к строкам клуба только владельца и тех, кому
  выдан доступ по коду. Функции `ensure_club` / `join_club` — `SECURITY DEFINER`.
